Mengão Monitor v3.5: Data Layer (Persistência Unificada)

2026-03-15 v3.5 python, sqlite, persistence, monitoring

O Problema

O Mengão Monitor tinha dados espalhados por todo lado:

  • UptimeHistory — SQLite separado pra histórico de checks
  • SLAReporter — dados em memória, perdidos no restart
  • AlertEscalation — alertas ativos em memória
  • Metrics — Prometheus scraping, sem query interna

Cada módulo reinventava a roda. Alguns persistiam, outros não. Queries cruzadas eram impossíveis.

A Solução: Data Layer Unificado

Uma camada SQLite única com 5 tabelas:

health_checks → histórico de verificações
alerts        → alertas com escalação
metrics       → time-series de métricas
incidents     → incidentes com MTTR
system_state  → key-value store

Features

  • Thread-safe — WAL mode + connection pooling por thread
  • Queries tipadas — QueryFilter com validação de SQL injection
  • Aggregações — avg, sum, min, max, count nativos
  • Percentis — p50, p95, p99 de response time
  • MTTR automático — calculado na resolução do incidente
  • Zero dependências — apenas stdlib (sqlite3, json, threading)

API Simples

from data_layer import DataLayer

dl = DataLayer(db_path="mengao_monitor.db")

# Health check
dl.record_check("api", "https://api.com", "up", response_time_ms=150)
uptime = dl.get_uptime("api", hours=24)  # 99.95

# Alert com escalação
dl.record_alert("alert_001", "api", "L1", "API down")
dl.escalate_alert("alert_001", "L2")
dl.update_alert_status("alert_001", "resolved")

# Métricas com agregação
dl.record_metric("response_time", 150.5)
avg = dl.get_metric_aggregate("response_time", "avg", hours=24)

# Incident com MTTR
dl.record_incident("inc_001", "api", "high", "Outage")
dl.resolve_incident("inc_001", "Fixed", root_cause="Memory leak")
mttr = dl.get_mttr()  # segundos

Integração com Health.py

13 novos endpoints /data/*:

  • GET /data/stats — estatísticas do banco
  • GET /data/checks — busca checks persistidos
  • GET /data/checks/uptime/{api} — uptime + response time
  • GET /data/alerts — busca alertas
  • GET /data/metrics?name=X — métricas time-series
  • GET /data/metrics/aggregate?name=X&agg=avg — agregações
  • GET /data/incidents — lista incidentes
  • POST /data/incidents — cria incidente
  • POST /data/incidents/{id}/resolve — resolve
  • GET /data/mttr — MTTR médio
  • POST /data/cleanup — limpa dados antigos (admin)
  • POST /data/vacuum — VACUUM no banco (admin)

Integração com Main.py

Cada check cycle agora persiste automaticamente:

# Em run_check_cycle():
dl = get_data_layer()
dl.record_check(
    api_name=endpoint.name,
    url=result["url"],
    status=result["status"],
    response_time_ms=result["response_time_ms"],
    status_code=result["status_code"],
    error=result.get("error")
)

Fallback silencioso se o data layer falhar — não quebra o monitor.
