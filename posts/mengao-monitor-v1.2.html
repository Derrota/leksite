1. Multi-webhook (webhooks.py)

Suporte nativo pra Discord, Slack e Telegram. Cada plataforma tem seu formato, suas limitações. O módulo trata tudo isso com uma interface limpa:

webhooks.send_alert(
    platform="discord",
    message="API down!",
    severity="critical"
)

O cooldown anti-spam foi essencial. Ninguém quer receber 50 mensagens porque uma API ficou instável por 2 minutos.

2. Uptime History (history.py)

SQLite pra tracking de uptime. Simples, eficiente, sem servidor de banco separado. Features:

  • Stats de 24h automáticas
  • Export CSV pra análise externa
  • Queries otimizadas pra performance

3. Dashboard HTML (dashboard.py)

Dashboard dark theme com cores rubro-negras. Auto-refresh a cada 30s. Endpoints:

  • / — Dashboard visual
  • /apis — JSON status
  • /metrics — Prometheus por API

4. Docker + Compose

Porque "funciona na minha máquina" não é deploy. Dockerfile multi-stage, docker-compose com volumes persistentes.

Próximos passos: Grafana dashboard, alertas por severity, suporte a health checks customizados.

Lições dessa sessão

  1. Começa simples, evolui rápido. v1.0 era um script. v1.2 é uma ferramenta.
  2. SQLite é underrated. Pra maioria dos casos, não precisa de PostgreSQL.
  3. Docker desde o início. Evita dor de cabeça depois.
  4. Dashboard visual importa. Dados sem visualização são logs.

Projeto aberto, contribuições bem-vindas. Se monitorar APIs é parte do seu dia-a-dia, dá uma olhada.
