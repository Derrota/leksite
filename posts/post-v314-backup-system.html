Mengão Monitor v3.14: Backup System

18 de Março de 2026

🔥 BACKUP AUTOMÁTICO PARA SQLITE

Full backup, compressão gzip, verificação SHA256 e rotação automática

O Backup System v3.14 é a camada de proteção final do Mengão Monitor. Enquanto o Maintenance System (v3.13) limpa dados antigos, o Backup System garante que nada importante se perca. Seu banco SQLite tem backup automático com compressão e verificação de integridade.

Por que Backup Automático?

SQLite é confiável, mas não é imune a:

  • Corrupção de disco — Falha de hardware às 3h da manhã
  • Erro humano — DELETE FROM health_checks sem WHERE
  • Deploy falho — Migração quebrada, dados perdidos
  • Ransomware — Arquivos criptografados, operação parada

O v3.14 resolve isso com backup automático, compressão e verificação. Se algo der errado, você tem um restore a um comando de distância.

1. Backup Full com Compressão

Cópia completa do banco SQLite com compressão gzip automática. Bancos de 100MB viram ~10MB comprimidos (dados repetitivos comprimem bem). 

curl -X POST http://localhost:8080/backup/run/monitor_db \
  -H "Authorization: Bearer mm_token"

2. Verificação de Integridade

Cria backup, descomprime temporariamente e executa PRAGMA integrity_check. Se o backup estiver corrompido, você sabe imediatamente — não quando precisar restaurar.

3. Rotação Automática

Duas políticas de retenção aplicadas automaticamente:

  • Por idade — Remove backups mais antigos que 30 dias
  • Por quantidade — Mantém apenas os 10 mais recentes

4. Restore com Verificação

Restaura backup e verifica integridade automaticamente. Se o banco restaurado estiver corrompido, remove e reporta erro. 

curl -X POST http://localhost:8080/backup/restore \
  -H "Authorization: Bearer mm_token" \
  -H "Content-Type: application/json" \
  -d '{
    "backup_path": "/var/backups/monitor_db_20260318_030000.db.gz",
    "restore_path": "/tmp/restored.db",
    "verify": true
  }'

Uso Programático

from backup import BackupManager, BackupConfig, BackupType

manager = BackupManager()

# Configurar backup
config = BackupConfig(
    name="monitor_db",
    source_path="mengao_monitor.db",
    backup_dir="/var/backups/mengao",
    backup_type=BackupType.FULL,
    compress=True,
    retention_days=30,
    max_backups=10,
    verify_after_backup=True
)

manager.add_config(config)

# Executar backup
result = manager.backup("monitor_db")
print(f"Status: {result.status.value}")
print(f"Size: {result.backup_size_bytes} bytes")
print(f"Compression: {result.compression_ratio:.1%}")

# Restaurar
success, error = manager.restore(
    backup_path=result.backup_path,
    restore_path="restored.db"
)

Arquitetura

O sistema é baseado em um BackupManager singleton que gerencia todas as configurações de backup. Cada backup é executado de forma síncrona (para garantir consistência) mas pode ser chamado de threads diferentes.

Fluxo de Backup

1. Verifica integridade do source (PRAGMA integrity_check)
2. Copia banco para arquivo temporário
3. Comprime com gzip (se configurado)
4. Calcula checksum SHA256
5. Verifica backup descomprimido (se configurado)
6. Move para diretório final
7. Remove backups antigos (rotação)
8. Registra no histórico

Lições da Implementação

"Backup sem verificação é esperança. Backup com verificação é garantia."

Algumas lições importantes:

  • SQLite é estável mas não perfeito — PRAGMA integrity_check é obrigatório antes de confiar no backup
  • Compressão gzip é essencial — SQLite comprime muito bem (dados repetitivos, índices, etc.)
  • Verificação pós-backup — Descomprimir e verificar é rápido e previne surpresas desagradáveis
  • Rotação automática — Sem ela, backups enchem o disco e causam o problema que deveriam prevenir
