Mengão Monitor v2.5: Plugin System

Arquitetura extensível para customizações infinitas

Depois do Circuit Breaker na v2.4, chegou a hora de tornar o Mengão Monitor verdadeiramente extensível. A v2.5 introduz um Plugin System completo que permite adicionar funcionalidades customizadas sem modificar o código core.

🔌 Por que Plugins?

O monitor já tinha muitas features built-in: webhooks, email, rate limiting, circuit breaker. Mas e se alguém quiser:

  • Enviar alertas via SMS ou PagerDuty?
  • Validar certificados SSL automaticamente?
  • Exportar métricas para DataDog ou Grafana Cloud?
  • Rodar testes de carga antes de alertar?

Com o Plugin System, tudo isso viram plugins de ~50 linhas que qualquer um pode criar.

🏗️ Arquitetura

Quatro tipos de plugins, cada um com responsabilidade clara:

PluginBase (abstract)
├── HealthCheckPlugin   → Custom health check logic
├── AlertHandlerPlugin  → Custom alert delivery
├── ExporterPlugin      → Export metrics externally
└── HookPlugin          → Run code on events

HealthCheckPlugin

Para quando o health check padrão não é suficiente:

class SSLCheckPlugin(HealthCheckPlugin):
    name = "ssl_check"
    
    def check(self, api_name: str, url: str, config: dict = None) -> dict:
        # Valida certificado SSL
        # Retorna status, dias restantes, issuer
        return {
            "status": "ok",
            "days_left": 45,
            "expires": "2026-05-01"
        }

AlertHandlerPlugin

Para levar alertas além do Discord/Slack:

class PagerDutyPlugin(AlertHandlerPlugin):
    name = "pagerduty"
    
    def send_alert(self, alert: dict) -> bool:
        # Integração com PagerDuty API
        return True

ExporterPlugin

Para mandar métricas pra onde quiser:

class DatadogExporter(ExporterPlugin):
    name = "datadog"
    
    def export(self, metrics: dict) -> bool:
        # Envia para DogStatsD
        return True

HookPlugin

Para executar código em momentos específicos:

class SlackNotifier(HookPlugin):
    def initialize(self, config):
        self.register_hook("startup", self.notify_startup)
        self.register_hook("alert_sent", self.log_alert)
    
    def notify_startup(self, **kwargs):
        # Notifica no Slack quando monitor inicia
        pass

📦 Plugin Manager

O PluginManager é o coração do sistema:

  • Registro — manual ou automático de diretórios
  • Descoberta — carrega todos os .py de um diretório
  • Ciclo de vida — initialize() e shutdown()
  • Configuração — cada plugin recebe seu próprio config
  • Enable/Disable — ativar/desativar sem remover

🧪 Testes

43 test cases cobrindo:

  • Registro e unregister de plugins
  • Execução de health checks
  • Envio de alertas
  • Exportação de métricas
  • Trigger de hooks
  • Load de diretórios
  • Tratamento de erros
  • Plugins desabilitados

🔌 Endpoints da API

GET  /plugins           → Lista todos os plugins
GET  /plugins/    → Detalhes de um plugin
POST /plugins//enable   → Habilita plugin
POST /plugins//disable  → Desabilita plugin
POST /plugins/load      → Carrega de diretório

📊 Exemplos Prontos

6 plugins de exemplo no plugins_examples/:
