Mengão Monitor v3.7: Performance Profiler

Medindo o que importa, encontrando o que dói

O Problema

36 versões. 8.000+ linhas. 200+ testes. Tudo funcionando. Mas uma pergunta ficava sem resposta: onde vai meu tempo?

Sabia que o health check era "rápido". Que o circuit breaker era "eficiente". Que o data layer era "otimizado". Mas números? Nenhum. Adivinhação com nome bonito.

A Solução: Performance Profiler

v3.7 adiciona um profiler integrado que mede tudo sem sujar o código de negócio. Dois modos de uso:

1. Decorator (para funções)

from performance_profiler import profile

@profile()
def check_endpoint(url):
    # código medido automaticamente
    pass

@profile("custom_name", metadata={"version": "3.7"})
def outra_funcao():
    pass

2. Context Manager (para blocos)

from performance_profiler import ProfileContext

with ProfileContext("data_layer_query"):
    results = db.execute(query)
    # só esse bloco é medido

O Que Ele Mede

  • Avg — tempo médio
  • P50 — mediana (50% das chamadas)
  • P95 — 95% das chamadas são mais rápidas que isso
  • P99 — 99% das chamadas são mais rápidas que isso
  • Min/Max — extremos
  • Std Dev — variabilidade

Detecção de Regressão

A feature mais útil: o profiler detecta automaticamente quando uma função fica mais lenta.

profiler = get_profiler()

# Threshold padrão: 20% de aumento
# Window: 100 amostras para baseline

regressions = profiler.get_regressions()
for r in regressions:
    print(f"{r.name}: +{r.regression_pct:.1f}% vs baseline")
    # "health_check_api: +35.2% vs baseline"

Ele compara as primeiras N amostras (baseline) com as últimas N (atual). Se a diferença passar do threshold, gera alerta.

Identificação de Gargalos

Dois endpoints que resolvem 90% dos problemas de performance:

  • /profiler/bottlenecks — ordena por tempo total (onde o sistema passa mais tempo)
  • /profiler/slowest — ordena por p95 (onde os outliers estão)

Relatório Automático

/profiler/report gera um JSON completo com:

  • Estatísticas globais
  • Top 10 gargalos
  • Top 10 funções mais lentas
  • Regressões detectadas
  • Recomendações automáticas

Thread Safety

Testado com 5 threads escrevendo e 3 lendo simultaneamente. Zero erros. Usa RLock para permitir reentrância.

Lições

  1. Meça antes de otimizar. 80% das "otimizações" são em código que não é gargalo.
  2. P95 > Avg. Média esconde outliers. P95 mostra o que o usuário sente.
  3. Regressão automática é essencial. Você não vai lembrar de comparar manualmente.
  4. Overhead zero é possível. if not enabled: return resolve.
