Mengão Monitor v1.1: Health Check, Testes e CLI

12 de março de 2026

Evoluí o Mengão Monitor pra v1.1. De um script simples de monitoramento pra uma ferramenta com health check HTTP, suite de testes, e CLI completa. Tudo isso mantendo a filosofia: simples, direto, sem overengineering.

O Que Mudou

🏥 Health Check Server (Flask)

Adicionei um servidor HTTP leve com Flask que expõe 3 endpoints:

Endpoint O que faz Formato
/health Status básico (up/down) JSON
/status Estado completo do monitor JSON
/metrics Métricas no formato Prometheus Text

Por quê Prometheus? Porque integra direto com Grafana. Se você já tem um stack de monitoramento, é só apontar pro endpoint e pronto. 

# Exemplo de uso
$ curl http://localhost:8080/health
{"status": "up", "uptime_seconds": 3600}

$ curl http://localhost:8080/metrics
# HELP mengao_monitor_checks_total Total de checks realizados
# TYPE mengao_monitor_checks_total counter
mengao_monitor_checks_total 42

🧪 Suite de Testes (pytest)

Nenhum projeto sério sem testes. Criei uma suite com pytest que cobre:

  • Carregamento de config — JSON válido, inválido, campos faltando
  • Health check — Flask endpoints respondendo corretamente
  • Monitor principal — State tracking, contadores, webhooks
  • Fixtures reutilizáveis — Config de exemplo, mock de requests
"Testes não são opcionais. São a diferença entre 'funciona na minha máquina' e 'funciona em produção'."

⌨️ CLI com argparse

Agora o monitor tem uma interface de linha de comando decente: 

# Uso básico
python monitor.py

# Com health check na porta 8080
python monitor.py --health --health-port 8080

# Config customizado
python monitor.py -c /path/to/config.json

Arquitetura

Mantive simples. Sem microservices, sem Kubernetes, sem firula: 

mengao-monitor/
├── monitor.py          # CLI + loop principal
├── health.py           # Flask health check server
├── test_monitor.py     # Suite pytest
├── requirements.txt    # Dependências
└── config.json         # Config de exemplo
