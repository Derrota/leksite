Mengão Monitor v3.9: Metrics Aggregator

📅 2026-03-16 ⏱️ 4 min leitura 🏷️ python, monitoring, metrics, anomaly-detection

Agregação temporal, detecção de anomalias via Z-Score, trend analysis e correlação de Pearson — tudo em 500 linhas de Python puro.

O Problema

O Mengão Monitor coleta métricas de APIs (response time, status codes, uptime). Mas dados brutos sem agregação são ruído. Precisava de:

  • Agregação em janelas: P95 de response time na última hora?
  • Detecção de anomalias: CPU disparou sem aviso?
  • Tendências: Latência subindo ao longo do dia?
  • Correlação: Quando CPU sobe, memory sobe junto?

A Solução: Metrics Aggregator

Módulo independente que agrega métricas em 5 janelas temporais e detecta anomalias automaticamente.

5 Janelas Temporais

class AggregationWindow(Enum):
    MINUTE_1 = 60
    MINUTE_5 = 300
    MINUTE_15 = 900
    HOUR_1 = 3600
    DAY_1 = 86400

Cada agregação calcula: count, min, max, mean, median, stddev, p95, p99, sum

Detecção de Anomalias (4 tipos)

  • SPIKE/DROP: Z-Score > 3.0 (configurável)
  • THRESHOLD: Ultrapassou min/max absoluto
  • FLATLINE: Valor não muda (possível erro de coleta)

Trend Analysis

Divide a janela em 4 quartis e compara médias:

  • rising: Todos os quartis crescem
  • falling: Todos os quartis decrescem
  • stable: Variação < 15% do stddev
  • volatile: Alternância de sinais

Correlação de Pearson

corr = agg.correlate("cpu_usage", "memory_usage")
# Resultado: {"correlation": 0.92, "strength": "strong", "direction": "positive"}

Uso Prático

from metrics_aggregator import get_aggregator, AggregationWindow, MetricThreshold

agg = get_aggregator()
agg.start()

# Registrar métricas
agg.record("response_time", 150.5)
agg.record("cpu_usage", 75.0)

# Agregar
result = agg.aggregate("response_time", AggregationWindow.MINUTE_1)
print(f"Mean: {result.mean}, P95: {result.p95}")

# Configurar threshold e detectar anomalias
agg.set_threshold(MetricThreshold("cpu_usage", max_val=90.0))
anomalies = agg.detect_anomalies("cpu_usage")

Lições
