Mengão Monitor v1.6: Rate Limiting e Retry Automático

Projeto 📅 2026-03-13 ⏱️ 5 min de leitura

Quando seu monitor de APIs detecta que algo caiu, o último problema que você quer é spam de alertas. O v1.6 do Mengão Monitor resolve isso com duas features que tavam na minha lista há tempo: rate limiting e retry automático.

O Problema

Imagina: sua API dá timeout por 5 segundos. Seu monitor detecta e manda alerta. Mas a API volta. E cai de novo. E volta. Em 1 minuto você recebeu 15 notificações no Discord.

⚠️ Alert spam é pior que não ter monitoramento. Quando tudo é urgente, nada é urgente.

O v1.6 resolve isso com camadas de proteção inteligentes.

🛡️ Rate Limiting: 5 Camadas de Proteção

O novo módulo rate_limiter.py implementa proteção contra spam em múltiplos níveis:

⏱️

Por Minuto

Máx 5 alertas/min por endpoint

📊

Por Hora

Máx 30 alertas/hora por endpoint

📅

Por Dia

Máx 100 alertas/dia por endpoint

💥

Burst

Máx 3 alertas em 60 segundos

❄️

Cooldown

5 min entre alertas do mesmo endpoint

Todas as camadas são thread-safe usando Lock, então funciona perfeitamente em produção com múltiplas threads.

🔄 Retry Automático com Backoff

O webhooks.py foi reescrito do zero. Agora, se um webhook falha, ele tenta de novo automaticamente:

Tentativa 1 → Falha (timeout) → Espera 1s
Tentativa 2 → Falha (503)     → Espera 3s  
Tentativa 3 → Sucesso (200)   → ✅ Alerta entregue

Regras Inteligentes

  • 3 tentativas por webhook
  • Backoff exponencial: 1s → 3s → 5s
  • Timeout de 10s por request
  • 4xx não retry — erro do cliente, não adianta tentar de novo
  • 5xx retry — erro do servidor, pode ser temporário

📊 Novas Métricas

O v1.6 adiciona métricas de Prometheus para monitorar o próprio monitor:

# Webhooks
mengao_webhook_sent_total 42
mengao_webhook_failed_total 3
mengao_webhook_retries_total 8

# Rate Limiting
mengao_rate_limit_allowed_total 156
mengao_rate_limit_blocked_total 12

Agora você pode criar alertas sobre seus alertas. Meta-monitoring! 🧠

O Que Aprendi

Rate limiting não é só sobre proteger APIs externas. É sobre proteger você mesmo do ruído. Um sistema que grita o tempo todo é tão inútil quanto um sistema silencioso.

Essa feature nasceu de uma dor real: receber 15 notificações porque uma API deu timeout de 2 segundos. Agora o monitor é inteligente o suficiente pra saber quando falar e quando calar a boca.
