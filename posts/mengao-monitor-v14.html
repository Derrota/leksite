Mengão Monitor v1.4: Alertas por Email com Templates HTML

13 Mar 2026 • 3 min leitura

Acabei de lançar a versão 1.4 do Mengão Monitor com uma feature que faltava: alertas por email via SMTP.

O que mudou

A v1.4 adiciona um módulo completo de notificações por email:

  • Templates HTML — Emails com design rubro-negro (claro, né?)
  • Fallback plain text — Para clientes de email mais simples
  • Cooldown anti-spam — Não bombardeia sua caixa de entrada
  • Suporte TLS — Conexão segura com o servidor SMTP
  • Eventos configuráveis — down, up, error
  • Test connection — Método para debugar configuração

Como funciona

O email_alerts.py usa uma classe EmailAlertSender que gerencia toda a lógica de envio:

from email_alerts import EmailConfig, EmailAlertSender

config = EmailConfig(
    smtp_host="smtp.gmail.com",
    smtp_port=587,
    username="seu-email@gmail.com",
    password="sua-app-password",
    from_addr="seu-email@gmail.com",
    to_addrs=["destino@email.com"]
)

sender = EmailAlertSender(config)
sender.send_alert("Minha API", "down", "Connection refused")

Integração com o monitor

Quando um endpoint muda de status (de up pra down ou vice-versa), o main.py agora dispara automaticamente o alerta por email se configurado:

"email": {
    "enabled": true,
    "smtp_host": "smtp.gmail.com",
    "smtp_port": 587,
    "username": "monitor@seudominio.com",
    "password": "app-password",
    "from_addr": "monitor@seudominio.com",
    "to_addrs": ["admin@seudominio.com"]
}
