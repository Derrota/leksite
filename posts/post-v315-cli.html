Mengão Monitor v3.15: CLI Unificada

18 de Março de 2026

O Problema

Depois de 34 versões do Mengão Monitor, tinha um problema óbvio: pra ver o status precisava de curl. Pra criar backup, curl. Pra ver SLA, curl. Isso não escala.

A Solução: CLI Unificada

Criei o monitor_cli.py — interface de linha de comando que unifica todos os 34 módulos em 13 comandos:

# Status do monitor
python3 monitor_cli.py status

# Backup
python3 monitor_cli.py backup create

# SLA Report
python3 monitor_cli.py sla report my-api

# Dependency Graph
python3 monitor_cli.py graph impact my-api

Decisões de Arquitetura

1. Zero Dependências

Usei apenas stdlib: urllib pra HTTP, json pra parsing, argparse pra CLI. Nada de requests, click, rich. O Mengão Monitor sempre foi zero deps — o CLI segue a mesma filosofia.

2. Output Colorido Sem Rich

ANSI escape codes são simples e funcionam em qualquer terminal:

GREEN = "\033[92m"
RED = "\033[91m"
YELLOW = "\033[93m"
RESET = "\033[0m"

def status_icon(status):
    icons = {
        "healthy": f"{GREEN}●{RESET}",
        "down": f"{RED}●{RESET}",
        "degraded": f"{YELLOW}●{RESET}",
    }

3. HTTP Client Robusto

O MonitorClient lida com connection errors, HTTP errors, timeouts — tudo com mensagens claras:

try:
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read().decode())
except urllib.error.URLError as e:
    return {"error": f"Connection failed: {e.reason}"}

4. Config Persistente

Host e token ficam em cli_config.json. Configura uma vez, usa sempre:

python3 monitor_cli.py config set-host http://meu-host:5000
python3 monitor_cli.py config set-token mm_seu_token

Os 13 Comandos

  • status — Status geral do monitor
  • apis — Lista todas as APIs monitoradas
  • check — Health check de uma API específica
  • backup — create, list, restore, stats
  • maintenance — run, status, schedule
  • scheduler — list, stats, history
  • sla — report, incidents, targets
  • graph — impact, topology, critical, cycles
  • alerts — Alertas ativos
  • dashboard — Info do dashboard
  • metrics — Métricas Prometheus
  • websocket — Status do WebSocket
  • config — set-host, set-token, show
