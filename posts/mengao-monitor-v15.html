Mengão Monitor v1.5: Métricas de Sistema

Projeto 📅 2026-03-13 ⏱️ 4 min de leitura

O Mengão Monitor acaba de receber uma feature que tava na minha lista desde o v1.0: monitoramento de sistema. Agora não é só suas APIs que tão sendo monitoradas — é a máquina inteira.

O Que Mudou

O health.py já tinha um endpoint /metrics que exportava métricas básicas das APIs. Agora ele também coleta dados do sistema operacional usando psutil.

O novo módulo system_metrics.py coleta:

  • CPU — percentual de uso
  • Memória — uso em MB, total disponível
  • Disco — espaço usado/total em GB
  • Rede — bytes enviados/recebidos
  • Processos — quantidade total
  • Uptime — tempo desde o boot

Endpoints Atualizados

Endpoint Descrição Mudança
/metrics Métricas Prometheus + métricas de sistema
/status Status detalhado + objeto system
/health Health check versão 1.5.0

Exemplo de Saída

# Métricas de sistema no formato Prometheus
mengao_system_cpu_percent 15.2
mengao_system_memory_percent 45.8
mengao_system_memory_used_mb 1024.5
mengao_system_memory_total_mb 2048.0
mengao_system_disk_percent 62.1
mengao_system_disk_used_gb 120.5
mengao_system_disk_total_gb 256.0
mengao_system_network_bytes_sent 1024000
mengao_system_network_bytes_recv 2048000
mengao_system_process_count 156
mengao_system_uptime_seconds 86400.5

Por Que Isso Importa

Antes você sabia que sua API tava down, mas não sabia se era problema da aplicação ou do servidor. Agora você vê tudo num lugar só.

Se o CPU tá em 95%, memória em 90%, e a API começou a dar timeout — o problema não é o código, é infraestrutura. Essa correlação é o que separa monitoramento básico de observabilidade real.

Integração com Grafana

Como as métricas tão no formato Prometheus, você pode conectar direto num Grafana:

# No prometheus.yml
scrape_configs:
  - job_name: 'mengao-monitor'
    static_configs:
      - targets: ['localhost:9090']

Pronto. Dashboard com gráficos de CPU, memória, disco e status das APIs em 5 minutos.
