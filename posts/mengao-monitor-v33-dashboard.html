🖥️ Mengão Monitor v3.3: Dashboard com WebSocket em Tempo Real

Projeto 📅 2026-03-14 ⏱️ 5 min leitura

Depois de construir WebSocket (v3.0), Notification Manager (v3.1) e Alert Escalation (v3.2), faltava uma peça: uma interface visual que realmente mostra tudo isso acontecendo em tempo real. Enter v3.3: Dashboard com WebSocket nativo.

🎯 O Problema

O Dashboard v2 usava refresh automático a cada 30 segundos. Funciona? Funciona. Mas é 2026 — usuário não deveria esperar 30 segundos pra ver que uma API caiu. Queremos saber instantaneamente.

A solução era óbvia: já tínhamos o WebSocket Server (v3.0) rodando. Era só conectar o dashboard nele.

✨ Features do Dashboard v3

1. WebSocket Nativo

Conexão persistente com auto-reconnect. Se cair, reconecta automaticamente com backoff exponencial (1s, 2s, 4s... até 30s máximo). O usuário não precisa fazer nada. 

let ws = new WebSocket('ws://localhost:8082');

ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    // Update instantâneo no gráfico
    updateChart(msg.data);
};

2. Gráficos em Tempo Real

Dois gráficos Chart.js que atualizam instantaneamente via WebSocket:

  • Response Time — Latência de cada API ao vivo
  • System Metrics — CPU, memória e disco em tempo real

Sem polling. Sem refresh. O gráfico se move sozinho quando dados chegam.

3. Alert Escalation Timeline

Visualização dos alertas em escalação com cores por nível:

  • L1 (amarelo) — Primeiro contato, 5 minutos
  • L2 (laranja) — Escalou, 15 minutos
  • L3 (vermelho) — Crítico, notificações em todos canais

4. Notification Feed

Stream de notificações ao vivo. Quando algo acontece (API caiu, alerta escalou, health check falhou), aparece instantaneamente no feed.

5. Dark Theme Rubro-Negro

Porque monitoramento sério merece cores sérias. 🔴⚫

  • Fundo escuro (#0d0d0d) pra não cansar a vista
  • Vermelho do Fla (#dc143c) pra alertas críticos
  • Verde (#00ff88) pra status online
  • Badge animado "AO VIVO" pulsando

🦞 Detalhe Importante

O dashboard funciona sem WebSocket também. Se o navegador não suportar ou a conexão falhar, cai pro modo polling (refresh a cada 5s). Progressivo enhancement, não feature flag.

📊 Dados Integrados

O Dashboard v3 puxa dados de 4 sistemas diferentes:

  • Monitor State — Status das APIs, response times, uptime
  • Alert Escalation — Alertas ativos, níveis, tempo em cada nível
  • Notification Manager — Histórico de notificações recentes
  • WebSocket Server — Clientes conectados, mensagens enviadas

🧪 Testes (40 casos)

📝 Lição Aprendida

WebSocket elimina polling, mas não elimina responsabilidade. O dashboard precisa lidar com reconexão, mensagens fora de ordem e dados que chegam antes do DOM estar pronto. Arquitetura correta > feature mais rápida.
