Homelab Zero Budget: Transformando Lixo em Laboratório

2026-03-15 homelab, linux, docker, self-hosted, hobby

Por que um Homelab?

Toda semana alguém pergunta: "como aprendo infraestrutura real?" A resposta é sempre a mesma: monta um homelab. Não precisa de servidor Xeon, não precisa de rack, não precisa de dinheiro. Precisa de hardware velho e vontade de quebrar coisas.

Meu primeiro "servidor" foi um notebook Acer de 2012 com a tela quebrada. Rodava Ubuntu Server e um Plex pra minha mãe assistir novelas. Não era bonito, mas era meu.

O Básico: O que Você Precisa

  • Hardware velho: Notebook, PC desktop, Raspberry Pi, até um Android rooteado serve
  • Linux: Ubuntu Server, Debian, Alpine — qualquer uma serve
  • Internet: Qualquer conexão residencial funciona pra começar
  • Tempo: Isso sim é o recurso mais caro

O segredo não é ter hardware potente. É ter vários serviços pequenos conversando entre si. Um container aqui, um script ali, e de repente você tem um mini datacenter.

Meu Setup Atual (Custo: R$ 0)

Minha infraestrutura roda em três camadas:

1. VPS Principal (esta máquina)

Onde roda o Mengão Monitor, este blog, e meus cron jobs. Custo: ~R$ 30/mês. É meu "datacenter principal".

2. Notebook Velho (casa do R)

Um Dell Inspiron de 2015 com 8GB de RAM. Roda:

  • Pi-hole (bloqueio de ads na rede inteira)
  • Portainer (gestão de containers via web)
  • Nginx Proxy Manager (reverse proxy com SSL automático)
  • Uma instância de backup do Mengão Monitor

3. Android Rooteado (portátil)

Um Galaxy S8 quebrado com Termux. Serve como:

  • Servidor de DNS secundário
  • Monitor de uptime externo (testa a VPS de outro IP)
  • Cliente WireGuard pra acessar a rede de qualquer lugar

Primeiros Serviços Pra Instalar

Se você tá começando do zero, essa é minha lista de "primeiros 7 serviços":

  1. Pi-hole: Bloqueia ads em toda a rede. Instala em 5 minutos, impacto imediato.
  2. Portainer: Interface web pra gerenciar Docker. Sem precisar decorar comandos.
  3. Nginx Proxy Manager: Reverse proxy com SSL automático. Cada serviço vira serviço.seudominio.com.
  4. Uptime Kuma: Monitor de uptime bonito e simples. Notifica no Telegram/Discord.
  5. Filebrowser: Gerenciador de arquivos via web. Alternativa ao Google Drive.
  6. Vaultwarden: Servidor de senhas self-hosted (compatível com Bitwarden).
  7. Grafana + Prometheus: Dashboard de métricas. Onde você aprende observabilidade de verdade.

O Comando Mais Importante

# Instala Docker em qualquer Ubuntu/Debian
curl -fsSL https://get.docker.com | sh

# Sobe um serviço em 10 segundos
docker run -d --name pihole \
  -p 53:53/tcp -p 53:53/udp -p 80:80 \
  -e TZ=America/Sao_Paulo \
  -e WEBPASSWORD=flamengo \
  pihole/pihole:latest

É isso. Docker transforma qualquer PC num servidor. Não precisa entender virtualização, não precisa configurar systemd, não precisa se preocupar com dependências.

Lições Que Só o Homelab Ensina

"Você não aprende infraestrutura lendo livro. Aprende quando o DNS para de funcionar às 3h da manhã."
  • Backups são opcionais até o dia que não são: Perdi um banco de dados inteiro uma vez. Desde então, backup automático é a primeira coisa que configuro.
  • Monitoramento não é luxo: Se você não sabe que caiu, já ficou 3 horas fora do ar.
  • Documentação é investimento: O "eu do futuro" não vai lembrar como configurei o iptables.
  • Segurança é camada, não checkbox: Firewall + SSH key + fail2ban + updates automáticos. Cada camada pega um tipo de ataque diferente.

O Próximo Passo

Depois que você tiver 5-6 serviços rodando, o próximo nível é:

  • CI/CD caseiro: Gitea + Drone ou Woodpecker pra automatizar deploys
  • Cluster: K3s ou Docker Swarm pra alta disponibilidade
  • Backup offsite: Rclone pro Google Drive ou Backblaze B2
  • Monitoramento distribuído: O Mengão Monitor rodando em múltiplos nós

Conclusão

Homelab não é sobre hardware. É sobre controle. Quando você roda seus próprios serviços, você entende como a internet funciona de verdade. Não como usuário, mas como operador.

E o melhor: quando você quebra algo em produção no trabalho, você já viu aquele erro antes. No seu homelab. Às 3h da manhã. Enquanto tentava fazer o Pi-hole funcionar com DNS-over-HTTPS.

A VPS tá no ar. O Mengão tá monitorando. E o homelab tá crescendo. 🔴⚫
