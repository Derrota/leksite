Mengão Monitor v3.6: Dependency Graph

2026-03-15 v3.6 python, monitoring, graph, dependencies

O Problema

Seu frontend caiu. Mas por quê? O backend tá up. O banco tá up. O cache tá up. Então por que o frontend não responde?

Resposta: o frontend depende de um serviço de auth que ninguém lembrava que existia. E esse serviço depende de um Redis que ninguém monitorava.

Esse é o problema que o Dependency Graph resolve: mapear dependências e calcular impacto de falhas.

O que é?

Um grafo de dependências entre endpoints monitorados. Cada nó é um serviço. Cada aresta é uma dependência. Quando algo cai, você sabe exatamente quem é impactado.

Features

  • Grafo de dependências: nós (endpoints) + arestas (dependências)
  • Análise de impacto: quantos serviços são afetados se X cair
  • Detecção de ciclos: identifica dependências circulares
  • Caminhos críticos: quais endpoints são mais críticos
  • Thread-safe: operações concorrentes seguras

Exemplo Prático

from dependency_graph import DependencyGraph

graph = DependencyGraph()

# Adiciona nós
graph.add_node("frontend", "https://frontend.com")
graph.add_node("api", "https://api.com")
graph.add_node("auth", "https://auth.com")
graph.add_node("db", "https://db.com")
graph.add_node("redis", "https://redis.com")

# Define dependências
graph.add_dependency("frontend", "api")
graph.add_dependency("frontend", "auth")
graph.add_dependency("api", "db")
graph.add_dependency("auth", "redis")

# Se o Redis cair, quem é impactado?
report = graph.calculate_impact("redis")
print(f"Impacto: {report.total_affected} serviços")
print(f"Direto: {report.direct_impact}")  # ['auth']
print(f"Transitivo: {report.transitive_impact}")  # ['frontend']
print(f"Severidade: {report.severity}")  # high

Endpoints da API

Método Endpoint Descrição
GET /graph/nodes Lista todos os nós
POST /graph/nodes Adiciona nó
POST /graph/dependencies Adiciona dependência
GET /graph/impact/<name> Impacto de falha
GET /graph/cycles Ciclos detectados
GET /graph/critical Endpoints por criticidade
GET /graph/topology Topologia para visualização

Análise de Impacto

Quando um endpoint cai, o grafo calcula:

  • Impacto direto: serviços que dependem diretamente do endpoint
  • Impacto transitivo: serviços afetados indiretamente
  • Severidade: low (0), medium (1-2), high (3-5), critical (6+)

Detecção de Ciclos

Dependências circulares são um problema real. A depende de B, B depende de C, C depende de A. O grafo detecta isso automaticamente.

graph.add_dependency("a", "b")
graph.add_dependency("b", "c")
graph.add_dependency("c", "a")  # ciclo!

cycles = graph.find_cycles()
print(f"Ciclos encontrados: {len(cycles)}")  # 1
