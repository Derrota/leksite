Mengão Monitor v3.8: Exporters (Exportação Multi-Destino)

16 de Março de 2026

📤 Métricas Sem Fronteiras

7 exporters, buffer inteligente, 57 testes — seus dados vão onde você quiser

O Mengão Monitor v3.8 resolve um problema real: para onde suas métricas vão? Não adianta coletar dados se eles ficam presos no monitor. Agora você exporta para Prometheus, Datadog, InfluxDB, Grafana, arquivos JSON/CSV, ou qualquer webhook genérico.

Os 7 Exporters

Cada exporter é uma classe independente, com validação própria e estatísticas separadas:

┌─────────────────┬─────────────────────────────────────┐
│ Exporter        │ Destino                             │
├─────────────────┼─────────────────────────────────────┤
│ Prometheus      │ Pushgateway (formato nativo)        │
│ Datadog         │ API v1/series (com tags)            │
│ InfluxDB        │ Line Protocol (séries temporais)    │
│ Grafana         │ Annotations (eventos visuais)       │
│ JSON File       │ Arquivo local (append/overwrite)    │
│ CSV File        │ Tabela com header automático        │
│ Webhook         │ POST genérico (qualquer endpoint)   │
└─────────────────┴─────────────────────────────────────┘

Buffer Inteligente

Em vez de exportar a cada métrica recebida, o sistema acumula num buffer e exporta em batch. Isso reduz overhead de rede e melhora throughput: 

# Buffer + flush periódico
manager.buffer_metrics(metrics)
manager.start_periodic_export(interval=60)  # flush a cada 60s

# Ou flush manual
results = manager.flush_buffer()

Filtros e Labels

Cada exporter pode filtrar métricas por prefixo e adicionar labels globais:

config = ExporterConfig(
    name="prod_json",
    labels={"env": "prod", "region": "us-east"},
    metric_filter=["cpu_", "memory_", "http_"]  # só exporta esses
)

API Completa

14 endpoints para gerenciar exporters via HTTP:

# Registrar novo exporter
curl -X POST /exporters/register -d '{
    "name": "influx_prod",
    "type": "influxdb",
    "config": {
        "url": "http://influx:8086",
        "database": "monitoring"
    }
}'

# Exportar métricas
curl -X POST /exporters/export-all -d '{
    "metrics": [
        {"name": "cpu_usage", "value": 75.5, "labels": {"host": "web1"}}
    ]
}'

# Ver histórico
curl /exporters/history?limit=50&status=success

Stats em Tempo Real

Cada exporter mantém estatísticas próprias:

  • total_exports — quantas vezes exportou
  • success_rate — % de sucesso
  • avg_duration_ms — tempo médio de exportação
  • total_metrics — total de métricas enviadas

Thread-Safe por Design

O ExporterManager usa RLock em todas as operações. Múltiplos threads podem exportar simultaneamente sem corrupção de dados. Testado com 5 threads × 10 exports = 50 operações concorrentes.

🔴⚫ Lição

Coletar métricas é fácil. O difícil é fazer elas chegarem onde precisam. O ExporterManager resolve isso com uma abstração simples: registra, bufferiza, exporta. Cada destino é um plugin independente — adicionar InfluxDB ou Datadog é trivial.
