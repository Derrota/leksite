Mengão Monitor v3.13: Automated Maintenance System

18 de Março de 2026

🔥 MANUTENÇÃO AUTOMÁTICA PARA SISTEMAS DE PRODUÇÃO

Log rotation, DB cleanup, cache eviction e disk monitoring em um único sistema

Um monitor de produção precisa monitorar a si mesmo. O Automated Maintenance System v3.13 é a camada de manutenção que garante que o Mengão Monitor não morra de morte lenta por falta de limpeza. Logs gigantes, banco inchado, cache obsoleto — tudo resolvido automaticamente.

Por que Manutenção Automática?

Depois de 39 versões em produção, aprendi que sistemas de monitoramento têm um paradoxo: eles monitoram tudo, menos a si mesmos. O resultado é previsível:

  • Logs de 10GB que enchem o disco às 3h da manhã
  • Banco SQLite de 500MB com dados de 6 meses atrás
  • Cache de health checks com resultados de APIs que não existem mais
  • Disk usage crítico sem alerta até ser tarde demais

O v3.13 resolve isso com um sistema de manutenção que roda em background, executando tarefas agendadas e reportando resultados.

Tipos de Tarefa

O sistema suporta 6 tipos de tarefa de manutenção:

LOG Rotation
DB Cleanup
CACHE Eviction
DISK Monitor

1. Log Rotation

Comprime logs antigos e remove arquivos que excedem o limite configurável. Suporta compressão gzip e rotação baseada em tamanho ou idade. 

curl -X POST http://localhost:8080/maintenance/tasks/log_rotation/run \
  -H "Authorization: Bearer mm_token"

2. Database Cleanup

Remove registros antigos de tabelas específicas. Usa DELETE com LIMIT para não sobrecarregar o banco durante a limpeza.

3. Cache Eviction

Remove entradas de cache expiradas ou que atingiram o limite de tamanho. Essencial para health checks que monitoram APIs dinâmicas.

4. Disk Monitoring

Monitora uso de disco e dispara alertas quando atinge thresholds configuráveis. Pode executar ações automáticas como limpeza de temporários.

Arquitetura

O sistema é baseado em um MaintenanceManager singleton que gerencia todas as tarefas. Cada tarefa é executada por um worker thread daemon, com histórico completo e callbacks para integração. 

from maintenance import MaintenanceManager, MaintenanceTask

manager = MaintenanceManager()

# Configurar log rotation
manager.setup_log_rotation(
    name="app_logs",
    log_dir="/var/log/mengao-monitor",
    max_size_mb=100,
    max_files=10,
    compress=True
)

# Configurar DB cleanup
manager.setup_db_cleanup(
    name="health_checks_old",
    db_path="monitor.db",
    table="health_checks",
    retention_days=30,
    batch_size=1000
)

Lições da Implementação

"Manutenção não é um custo — é um investimento contra o caos futuro."

Algumas lições importantes:

  • DELETE com LIMIT: SQLite não suporta LIMIT em DELETE. Solução: SELECT com LIMIT para obter IDs, depois DELETE WHERE id IN (...)
  • Thread safety: Cada tarefa roda em thread separada, mas o manager usa RLock para coordenar acesso
  • Callbacks são opcionais: Sistema funciona sem callbacks, mas com eles fica muito mais poderoso
  • Zero dependências: Tudo com stdlib — shutil, sqlite3, threading
