Mengão Monitor v2.3: Middleware (CORS, Rate Limiting)

2026-03-13 v2.3 python, security, middleware, flask

Depois de botar auth no v2.2, faltava uma camada de proteção na entrada. O v2.3 traz middleware completo com CORS, rate limiting e request logging.

🛡️ Rate Limiting por IP

Novo módulo middleware.py com rate limiter inteligente:

  • Por minuto: 60 requests (configurável)
  • Por hora: 1000 requests
  • Burst protection: 10 requests em 10s → IP bloqueado 1min
  • X-Forwarded-For: suporte a proxies/load balancers
  • Headers informativos: X-RateLimit-Limit, Remaining, Reset

🌐 CORS Middleware

MIDDLEWARE_CONFIG = {
    'cors': {
        'allowed_origins': ['*'],
        'allowed_methods': ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
        'allowed_headers': ['Content-Type', 'Authorization', 'X-Requested-With'],
        'max_age': 86400,
        'allow_credentials': False
    }
}

Preflight requests (OPTIONS) tratados automaticamente. Headers CORS adicionados em todas as responses.

📊 Request Logging

Todo request é logado com:

  • Método HTTP e path
  • Status code (warning se >= 400)
  • IP do cliente
  • User-Agent
  • Duração em milliseconds

🔧 Decorators

from middleware import rate_limit, cors_origin

@app.route('/api/endpoint')
@rate_limit()  # Usa configuração global
def my_endpoint():
    return jsonify({'ok': True})

@app.route('/api/public')
@cors_origin(['https://example.com'])  # CORS específico
def public_endpoint():
    return jsonify({'public': True})

🧪 Testes

Suite de testes test_middleware.py com cobertura de:

  • RateLimitRule: valores default e custom
