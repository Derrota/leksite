Mengão Monitor v2.1: API REST + Hot-reload de Configuração

2026-03-13 v2.1 python, api, flask, monitoring

Depois do v2.0 com dashboard Chart.js, era hora de evoluir pra algo que todo monitor de verdade precisa: gerenciar endpoints sem restart.

🔧 API REST de Gerenciamento

O novo módulo api_manager.py expõe uma API REST completa em :8081/api/v1/:

# Listar endpoints
curl http://localhost:8081/api/v1/endpoints

# Adicionar em runtime
curl -X POST http://localhost:8081/api/v1/endpoints \
  -H "Content-Type: application/json" \
  -d '{"name": "Nova API", "url": "https://api.com/health"}'

# Pausar sem remover
curl -X POST http://localhost:8081/api/v1/endpoints/Nova%20API/pause

# Retomar
curl -X POST http://localhost:8081/api/v1/endpoints/Nova%20API/resume

👁️ Hot-reload de Configuração

O config_watcher.py monitora o config.json a cada 10 segundos. Quando detecta mudança (via hash SHA256), recarrega automaticamente e loga o diff:

🔄 Config change detected: config.json
🔄 Hot-reload: Processing config changes...
  Added: 1 endpoints
  Removed: 0 endpoints
  Modified: 2 endpoints
✅ Config reloaded successfully

🏗️ Arquitetura Thread-safe

Os dois módulos usam threading.Lock pra garantir segurança em ambiente concorrente. O API Manager roda em thread separada (Flask daemon), e o Config Watcher também. Callbacks conectam tudo ao monitor principal.
